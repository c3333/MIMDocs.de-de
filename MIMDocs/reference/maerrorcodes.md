# <a name="management-agent-run-error-codes"></a>Fehlercodes für den Verwaltungs-Agent

Die folgenden Tabellen enthalten Fehlercodes, die in Microsoft Identity Manager (MIM) 2016 in der Synchronization Service Manager-Benutzeroberfläche angezeigt werden könnten, sowie Beschreibungen für jeden dieser Fehler.

## <a name="connection-errors"></a>Verbindungsfehler

| Fehler                  | Beschreibung                                    |
|--------------------------|--------------------------------------------|
| failed-connection         | Bei der Verbindung mit dem verbundenen Verzeichnis ist aus einem anderen Grund als der Authentifizierung ein Fehler aufgetreten. Beispielsweise ist das Netzwerk nicht verfügbar oder der Zielserver offline.                                                                                                                                                                                                                                                                                                                      |
| dropped-connection        | Die Verbindung zwischen dem Verwaltungs-Agent und dem verbundenen Verzeichnis ist nicht mehr vorhanden. Der Verwaltungs-Agent versucht in vielen Fällen, erneut eine Verbindung mit dem verbundenen Verzeichnis herzustellen.                                                                                                                                                                                                                                                                                                         |
| failed-authentication     | Authentifizierung ist mithilfe der angegebenen Anmeldeinformationen nicht möglich.                                                                                                                                                                                                                                                                                                                                                                                                                          |
| failed-permission         | Unzureichende Berechtigungen zum Zugriff auf einen Container im verbundenen Verzeichnis. Dieser Fehler kann nur bei Verwaltungs-Agents des Lightweight Directory Access Protocol (LDAP) auftreten, die anders verbundene Directory-Container durchsuchen.                                                                                                                                                                                                                                                              |
| failed-search            | Beim Durchsuchen eines Containers oder einer Tabelle ist ein Fehler aufgetreten.                                                                                                                                                                                                                                                                                                                                                                                                                            |
| warning-no-watermark      | Der Verwaltungs-Agent kann das Wasserzeichen bei einem vollständigen Import nicht lesen. Dieser Fehler kann nur bei Verwaltungs-Agents für Sun ONE Directory Server 5.1 (ehemals iPlanet Directory Server) auftreten, wenn die anfängliche Verwaltungs-Agent-Konfiguration abgeschlossen und für das verbundene Verzeichnis das Änderungsprotokoll aktiviert ist. Wenn später das Änderungsprotokoll des verbundenen Verzeichnisses ausgeschaltet ist, und die Konfiguration des Verwaltungs-Agents nicht aktualisiert wird, wird diese Warnung bei einem vollständigen Import ausgegeben. |
| no-start-partition-delete | Die ursprünglich konfigurierte LDAP-Partition für diesen Verwaltungs-Agent ist nicht mehr vorhanden. Dieser Fehler wird zurückgegeben, wenn die Partition gelöscht wurde, bzw. wenn die Partition gelöscht und mit dem gleichen Namen neu erstellt wurde. In letzterem Fall wird die Fehlerpartitions-GUID geändert, obwohl die Namen übereinstimmen.                                                                                                                                                                           |

## <a name="discovery-errors"></a>Ermittlungsfehler

| Fehler                   | Beschreibung                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
|-------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| missing-change-type      | Dieser Fehler wird sowohl während eines von dateibasierten und Datenbankverwaltungs-Agents ausgeführten Deltaimports zurückgegeben, als auch vom Verwaltungs-Agent für Sun- und Netscape-Verzeichnisserver, wenn der Änderungstyp-Spaltenwert („Hinzufügen“, „Ändern“, „Löschen“) nicht vorhanden ist.                                                                                                                                                                                                                                          |
| invalid-change-type      | Dieser Fehler wird sowohl während eines von dateibasierten und Datenbankverwaltungs-Agents ausgeführten Deltaimports zurückgegeben, als auch vom Verwaltungs-Agent für Sun- und Netscape-Verzeichnisserver, wenn der Änderungstyp-Spaltenwert nicht mit der Liste gültiger Änderungstypen übereinstimmt. Er wird auch von einem vollständigen LDAP Datenaustauschformat-Import (LDAP Data Interchange Format, LDIF) zurückgegeben, wenn ein Änderungstypfeld vorhanden ist und einen anderen Wert als „Hinzufügen“ hat.                                                                                  |
| multi-valued-change-type | Dieser Fehler wird während eines von dateibasierten Verwaltungs-Agents und Verwaltungs-Agents für Sun- und Netscape-Verzeichnisserver ausgeführten Deltaimports zurückgegeben, wenn mehrere Werte für den Änderungstyp vorhanden sind.                                                                                                                                                                                                                                                                                                        |
| need-full-object        | Dieser Fehler wird während eines von dateibasierten Verwaltungs-Agents ausgeführten Deltaimports oder beim Fortsetzen von einem dateibasierten Verwaltungs-Agent aus zurückgegeben. Er gibt an, dass der Verwaltungs-Agent eine Änderung an einem Objekt übermittelt hat, das im Connectorbereich nicht gefunden werden kann. Das Synchronisierungsmodul fordert die aktuellen Werte aller Attribute für das Objekt an. Da es sich um einen Import aus einer Datei handelt, sind diese Informationen nicht verfügbar. Ein vollständiger Import sollte dieses Problem beheben. |
| missing-dn              | Dieser Fehler wird für dateibasierte Verwaltungs-Agents (d. h. Verwaltungs-Agents für LDIF, DSML oder Flatfiles mit konfigurierten Domänennamensattributen) zurückgegeben, wenn kein Domänennamenswert vorhanden ist. Er wird auch bei einem beschädigten Sun ONE-Verzeichnisserver-Änderungsprotokoll zurückgegeben, in dem das Domänennamensattribut fehlt. Er gibt an, dass der Verwaltungs-Agent das Element lesen und analysieren konnte, aber kein Domänennamenswert für das Objekt vorhanden war.                           |
| dn-not-ldap-conformant   | Dieser Fehler wird zurückgegeben, wenn ein Verwaltungs-Agent für LDAP, LDIF, DSML oder eine Flatfile mit einem konfigurierten Domänennamensattribut einen Domänennamenswert meldet, der nicht der LDAP-Spezifikation entspricht.                                                                                                                                                                                                                                                                                |
| invalid-dn              | Dieser Fehler wird zurückgegeben, wenn ein Verwaltungs-Agent meldet, dass ein Domänenname einer FIM-Einschränkung nicht entspricht, was beinhaltet:   <br/>&#176; Mindestens ein Zeichen, das nicht bei FIM zulässig ist <br/>&#176; Ein leerer Relative Distinguished Name (RDN) <br/>&#176; Ein Relative Distinguished Name, der das Maximum für FIM überschreitet <br/>&#176; Die Anzahl der Hierarchieebenen des Domänennamens überschreitet das Maximum für FIM                                                                                                                                                                                                                                                                                                                                                            |
| missing-anchor-component | Dieser Fehler wird sowohl durch dateibasierte und Datenbank-Verwaltungs-Agents als auch den Verwaltungs-Agent für Sun- und Netscape-Verzeichnisserver zurückgegeben, wenn der Anker nicht konstruiert werden kann, da mindestens ein Ankerkonstruktionsregel-Attribut keinen Wert hat.   |
| multi-valued-anchor-component | Dieser Fehler wird durch den Verwaltungs-Agent für Sun- und Netscape-Verzeichnisserver zurückgegeben, wenn sie den Anker nicht konstruieren können, da ein Ankerkonstruktionsregel-Attribut mehrere Werte hat. |   
|anchor-too-long | Dieser Fehler wird sowohl von dateibasierten und Datenbankverwaltungs-Agents als auch vom Verwaltungs-Agent für Sun- und Netscape-Verzeichnisserver zurückgegeben, wenn die Ankerkonstruktion einen Anker hervorbringt, der die FIM-Maximalgröße überschreitet.|
|duplicate-object | Dieser Fehler wird bei vollständigen Importen durch dateibasierte und Datenbank-Verwaltungs-Agents zurückgegeben, wenn während dieser Ausführung bereits ein Objekt mit dem gleichen Anker an das Synchronisierungsmodul gemeldet wurde.|

>[!Note]
Überalterung von Connectorbereichsobjekten tritt nur dann auf, wenn der aktuelle Ausführungsschritt mit „Erfolg“, „Abgeschlossen mit Synchronisierungsfehler“, „Abgeschlossen mit Warnung abgeschlossen“ oder „Vorübergehend abgeschlossen“ abgeschlossen wurde.

Er gibt an, dass der Verwaltungs-Agent einen Wert für das Objektklassenattribut nicht lesen kann. |

Der Server erstellt ein Ereignisprotokoll, das Informationen zum Problem bietet und zur Problembehandlung verwendet werden kann. Die meisten Verwaltungs-Agents setzen die Importausführung fort, wenn der Fehler protokolliert wird, aber der Verwaltungs-Agent für Sun- und Netscape-Deltaausführungen stoppt, da Lücken in der Änderungsprotokollverarbeitung einen inkonsistenten Zustand im Connectorbereich verursachen könnten. Dieser Fehler sollte während des normalen Betriebs nicht auftreten.  |
|invalid-modification-type | Dieser Fehler wird während eines Deltaimports auf einem LDIF-Verwaltungs-Agent zurückgegeben, wenn ein Änderungstyp auf Objektebene nicht einer der standardmäßigen LDIF-Änderungstypen ist, oder ein Nonreplace-Änderungstyp in der Objektklasse vorhanden ist, z.B. „add: objectclass“ oder „delete: objectclass“.    |
|conflicting-modification-types        | Dieser Fehler wird vom LDIF-Verwaltungs-Agent zurückgegeben und gibt an, dass im gleichen Datensatz unterschiedliche Änderungstypen auf Attributebene aufgetreten sind (in diesem Fall wird der Attributname gemeldet, der die in Konflikt stehenden Typen erzeugte), oder dass mehrere Ersatz-LDIF-Deltas in der gleichen Datei erkannt werden, z.B.:    <br/> [LDIF-Dateibeispiel](./media/maerrorcodes/conflict-modification-types.jpg) |
|multi-single-mismatch | Dieser Fehler wird durch einen dateibasierten Verwaltungs-Agent zurückgegeben, wenn er mehrere add- oder delete-Werte für ein Attribut meldet, das in FIM als Einzelwertattribut definiert ist. Dieser Fehler kann darauf hinweisen, dass das Schema der verbundenen Datenquelle, das mit FIM gespeichert ist, falsch angegeben (dateibasierte Verwaltungs-Agents) oder im Vergleich zum aktuellen Schema veraltet ist. Enthält ein `<attribute-name>`-Element, um den Kontext des Fehlers anzugeben.    |
|invalid-attribute-value | Dieser Fehler wird durch aufrufbasierte Verwaltungs-Agents zurückgegeben, wenn ein Attributwert gelesen wird, der nicht dem im Schema deklarierten Attributtyp entspricht. Enthält ein `<attribute-name>`-Element, um den Kontext des Fehlers anzugeben.  |
|invalid-base64-value| Dieser Fehler wird durch die Verwaltungs-Agents für LDIF, DSML und Sun- und Netscape-Verzeichnisserver zurückgegeben, wenn sie eine ungültige base64-Zeichenfolge erkennen.     |
|invalid-numeric-value | Dieser Fehler wird durch dateibasierte Verwaltungs-Agents und den Verwaltungs-Agent für LDAP zurückgegeben, wenn sie einen numerischen Wert nicht analysieren können. Enthält ein `<attribute-name>`-Element, um den Kontext des Fehlers anzugeben. |
|invalid-boolean-value | Dieser Fehler wird durch dateibasierte Verwaltungs-Agents und den Verwaltungs-Agent für LDAP zurückgegeben, wenn sie einen booleschen Wert nicht analysieren können. Enthält ein `<attribute-name>`-Element, um den Kontext des Fehlers anzugeben.  |
|reference-value-not-ldap-conformant | Dieser Fehler wird von Verwaltungs-Agents für LDAP, LDIF und DSML oder Flatfiles (mit konfiguriertem Domänennamensattribut) zurückgegeben, wenn ein Domänennamenswert nicht der LDAP-Spezifikation entspricht. Diese Fehlermeldung enthält ein `<attribute-name>`-Element, um den Kontext des Fehlers anzugeben.|
|invalid-reference-value | Dieser Fehler wird von einem Verwaltungs-Agent zurückgegeben, wenn ein Domänenname nicht FIM-Einschränkungen entspricht – hierzu zählen:  <br/>&#176; Mindestens ein Zeichen, das nicht bei FIM zulässig ist   <br/>&#176; Ein leerer Relative Distinguished Name (RDN)  <br/>&#176; Ein Relative Distinguished Name, der das Maximum für FIM überschreitet  <br/>&#176; Die Anzahl der Hierarchieebenen des Domänennamens überschreitet das Maximum für FIM                |
|unsupported-value-type | Dieser Fehler wird durch den DSML- oder LDIF-Verwaltungs-Agent zurückgegeben, wenn der in der Datei angegebene Werttyp nicht mit dem Typ des Attributs kompatibel ist, einschließlich:    <br/>&#176; Ein URI- oder URL-Wert wird für ein Nichtzeichenfolgen-Attribut oder ein beliebiges reserviertes Schlüsselwort wie „dn“, „objectclass“ oder „changetype“ angegeben.  <br/>&#176; Für das changetype-Attribut wird ein base64-Wert angegeben.  <br/>&#176; Ein Zeichenfolgenwert, der Nicht-ASCII-Zeichen enthält, wird für ein binäres Attribut angegeben. |

## <a name="synchronization-errors"></a>Synchronisierungsfehler

| Fehler               |Beschreibung      |
|---------|-------------|
| extension-dll-exception  | Dieser Fehler tritt auf, wenn eine Regelerweiterung eine Ausnahme verursacht. Wenn dieser Fehler auftritt, untersuchen Sie die Aufrufliste der Ausnahme anhand des \<exception-error-info\>-Elements. In einigen Fällen ist die `<rule-error-info>` vorhanden und bietet zusätzliche Informationen dazu, welche Regel verarbeitet wurde, als der Fehler auftrat.        |
| extension-dll-crash | Dieser Fehler tritt auf, wenn der Prozess, der die Regelerweiterung ausführt, unerwartet beendet wird. Dieser Fehler kann nur auftreten, wenn eine Regelerweiterung außerhalb des Prozesses ausgeführt wird. Eine mögliche Ursache für diesen Fehler ist, dass die Regelerweiterung Code aufruft, der eine Zugriffsverletzung verursacht. |
| extension-dll-timeout                          | Dieser Fehler tritt auf, wenn der Kunde ein Timeout für die Erweiterung konfiguriert hat, und der Aufruf an einem einzelnen Kundenerweiterungs-Codeeinstiegspunkt das konfigurierte Timeout überschreitet. Die `<exception-error-info>` bietet Kontextinformationen dazu, welcher Einstiegspunkt aufgerufen wurde, als das Timeout in Kraft trat. In einigen Fällen ist die <rule-error-info> vorhanden und bietet zusätzliche Informationen dazu, welche Regel verarbeitet wurde, als der Fehler auftrat. Beachten Sie, dass beim Debuggen des Prozesses, der die Erweiterung ausführt, keine Timeouts erzwungen werden.  |              
| extension-projection-object-type-not-set        | Dieser Fehler tritt auf, wenn die Implementierung der **IMASynchronization.ShouldProjectToMV**-Methode in der Regelerweiterung nicht den Metaverse-Objekttyp angibt.                    |
| extension-projection-invalid-object-type        | Dieser Fehler tritt auf, wenn die Implementierung der **IMASynchronization.ShouldProjectToMV**-Methode in der Regelerweiterung den Wert des ausgehenden Metaverse-Objekttyps auf einen Wert festlegt, der im Metaverse-Designer von Synchronization Service Manager nicht aufgelistet ist. Überprüfen Sie, ob die Methode einen der angegebenen Objekttypwerte verwendet.   |
| extension-join-resolution-invalid-object-type   | Dieser Fehler tritt auf, wenn die Implementierung der **IMASynchronization.ResolveJoinSearch**-Methode in der Regelerweiterung den Wert des ausgehenden Metaverse-Objekttyps auf einen Wert festlegt, der im Metaverse-Designer von Synchronization Service Manager nicht aufgelistet ist. Überprüfen Sie, ob die Methode den Wert des ausgehenden Metaverse-Objekttyps auf einen der aufgelisteten Objekttypwerte festlegt.   |
| extension-join-resolution-index-out-of-bounds   | Dieser Fehler tritt auf, wenn eine Implementierung der **IMASynchronization.ResolveJoinSearch**-Methode in der Regelerweiterung einen Indexwert festlegt, der entweder negativ oder größer als die Anzahl der Metaverse-Objekte ist.                     |
| extension-provisioning-call-limit-reached       | Dieser Fehler tritt auf, wenn die **IMASynchronization.Provision**-Methode während der Synchronisierung eines einzelnen Objekts mehr als 10 Mal aufgerufen wird. Diese Methode kann mehrmals aufgerufen werden, wenn die Kundenlogik in der Provision-Methode die Bereitstellung eines Objekts aufhebt und ein erneuter Attributaufruf eine Änderung des Metaverse-Objekts bewirkt, woraus ein neuer Provision-Aufruf resultiert. Der Grenzwert von 10 Aufrufen für die Provision-Methode soll mögliche Hinweise zu unbegrenzter Bereitstellung stoppen.                                                                                                                                               |
| extension-deprovisioning-invalid-result         | Dieser Fehler tritt auf, wenn eine Implementierung der **IMASynchronization.Deprovision**-Methode einen ungültigen DeprovisionAction-Enumerationswert zurückgibt. Stellen Sie sicher, dass die Methode einen gültigen Wert zurückgibt.        |
| extension-entry-point-not-implemented           | Dieser Fehler tritt auf, wenn eine Regelerweiterung eine **EntryPointNotImplementedException**-Ausnahme auslöst. |
| extension-unexpected-attribute-value            | Dieser Fehler tritt auf, wenn eine Regelerweiterung eine **UnexpectedDataException**-Ausnahme auslöst.            |
| flow-multi-values-to-single-value              | Dieser Fehler tritt auf, wenn eine im Synchronization Service Manager konfigurierte Import- oder Exportattributfluss-Regel versucht, ein Attribut mit mehreren Werten in ein Einzelwertattribut einfließen zu lassen. Dieser Fehler wird nur für direkte Flussregeln zurückgegeben, die im Synchronization Service Manager konfiguriert werden. Wenn die Flussregel eine Regelerweiterung verwendet, die mehrere Werte in ein Einzelwertattribut einfließen lässt, wird die **TooManyValuesException**-Ausnahme ausgelöst.          |
| cs-attribute-type-mismatch                     | Dieser Fehler tritt auf, wenn der Typ des importierten Attributs nicht mit dem Typ des im Verwaltungs-Agent-Schema angegebenen Attributs übereinstimmt. Eine Ursache für diesen Fehler ist möglicherweise, dass das gespeicherte Schema der verbundenen Datenquelle im Vergleich zum aktuellen Schema der verbundenen Datenquelle veraltet ist. Um das gespeicherte Schema der verbundenen Datenquelle auf den neuesten Stand zu bringen, aktualisieren Sie das Schema mit dem Synchronization Service Manager.|
| join-object-id-must-be-single-valued            | Dieser Fehler tritt auf, wenn der Datenquellen-Attributwert, der zum Verknüpfen eines Metaverse-Objekts mittels einer Verknüpfungsregel verwendet wird, die in den Eigenschaften eines Verwaltungs-Agents in Synchronization Service Manager angegeben ist, mehr als einen Wert enthält. Der in der Verknüpfungsregel verwendete Datenquellen-Attributwert kann nur einen einzelnen Wert enthalten.     |
| dn-index-out-of-bounds       | Dieser Fehler tritt auf, wenn der Distinguished Name-Komponentenindexwert, der in einem Importattributfluss verwendet wird, der in den Eigenschaften eines Verwaltungs-Agents im Synchronization Service Manager konfiguriert ist, größer als die Anzahl der Komponenten im Distinguished Name des Quellobjekts ist.      |
| connector-filter-rule-violation                | Dieser Fehler tritt beim Ausführen eines Hinzufügungs- oder Umbenennungs-Bereitstellungsvorgangs oder beim Exportattributfluss auf, und wenn ein Connectorobjekt als Ergebnis einer Connectorfilterkonfiguration zu einem gefilterten Disconnectorobjekt wird. Dieser Wert tritt nicht bei expliziten Connectorobjekten auf.    |
| unsupported-container-delete   | Der Verwaltungs-Agent versucht, ein Containerobjekt während des Aufhebens der Bereitstellung zu löschen. FIM-Verwaltungs-Agents können keine Containerobjekte mit untergeordneten Objekten löschen.            |
| ambiguous-import-flow-from-multiple-connectors   | Dieser Fehler tritt auf, wenn sich mehrere Connectors unter dem Quellverwaltungs-Agent befinden, der mit dem Metaverse-Objekt verbunden ist, und eine deklarative Importattribut-Flussregel definiert ist. Zum Importieren von Attributen über einen Verwaltungs-Agent mit mehreren Connectors zu einem Metaverse-Objekt verwenden Sie eine Regelerweiterung, um die Flussregeln zu definieren, anstatt eine direkte Regel in den Eigenschaften eines Verwaltungs-Agents zu konfigurieren.                                                                                                                                                                                                          |
| ambiguous-export-flow-to-single-valued-attribute | Dieser Fehler tritt auf, wenn die in den Eigenschaften für einen Verwaltungs-Agent im Synchronization Service Manager konfigurierte Exportflussregel versucht, mehrere Werte aus einem Metaverse-Objekt in ein Einzelwertattribut einfließen zu lassen.       |
| cannot-parse-object-id | Der Zeichenfolgenwert, der für die Suche nach einem Metaverse-Objekt in einer Verknüpfungsregel verwendet wird, die in den Eigenschaften eines Verwaltungs-Agents im Synchronization Service Manager angegeben ist, liegt nicht im richtigen GUID-Format (Globally Unique Identifier) vor. Das GUID-Format ist {nnnnnnnn-nnnn-nnnnnnnn-nnnnnnnnnnnn}, wobei n eine hexadezimale Zahl ist. |
| unexported-container-rename  | Die Implementierung der IMVSynchronization.Provision- oder IMASynchronization.Deprovision-Methode versucht, ein Containerobjekt mit mindestens einem nicht exportierten untergeordneten Objekt umzubenennen.   |
| mv-constraint-violation           | Dieser Fehler tritt auf, wenn bei direktem Importattributfluss der Wert des Attributs aus dem Connectorbereich die Längeneinschränkungen für das Metaverse-Attribut überschreitet. |
| locking-error-needs-retry   | Mehrere Verwaltungs-Agents versuchen, das gleiche Connectorbereichsobjekt zu synchronisieren. Führen Sie den Verwaltungs-Agent erneut aus.     |
| unique-index-violation              | Ein Benutzer legt manuell einen eindeutigen Index für ein Attribut in einer Metaverse-Tabelle fest. Konfigurieren Sie die Metaverse-Tabellen nicht manuell.    |
| encryption-key-lost                            | Die Verschlüsselungsschlüsselgruppen fehlen auf dem Server, der FIM ausführt.  |
| unexpected-error                              | Dieser Fehler tritt auf, wenn das Synchronisierungsmodul versucht, eine Metaverse-Änderung (einschließlich Bereitstellung und Exportattributfluss) zu übernehmen. Dieser Fehler kann nur während Ausführungen auftreten, die Metaverse-Änderungen übernehmen. Prüfen Sie das Ereignisprotokoll, um weitere Informationen zu erhalten.  |
| exported-change-not-reimported                 | Dieser Fehler tritt auf, wenn Änderungen, die nach einem Verwaltungs-Agent exportiert werden, nicht während der Ausführung dieses Importverwaltungs-Agents erneut bestätigt werden. Ein Benutzer oder Systemprozess außerhalb von FIM hat die Daten in der verbundenen Datenquelle auf eine Weise geändert, die auf ein Konfigurationsproblem hinweist, wobei die Exportattributfluss-Regel versucht, einen Wert zu einem verbundenen Datenquellenobjekt fließen zu lassen, aber die verbundene Datenquelle automatisch einen anderen Wert festlegt, ohne dem Verwaltungs-Agent einen Fehler zu melden. Das \<change-not-reimported\>-Element gibt an, welche Änderungen nicht erneut bestätigt wurden. |
| cannot-parse-dn-component                      | Dieser Fehler wird durch einen Verwaltungs-Agent zurückgegeben, für den ein Distinguished Name (auch DN genannt) im LDAP-Format konfiguriert ist, und wo bei der Synchronisierung vom Connectorbereich aus mit Metaverse ein Fehler aufgetreten ist. Eine Distinguished Name-Komponente kann nicht durch eine dncomponent-Zuordnung analysiert werden, da sie nicht das richtige Format für den Zielattributtyp hat.|
| missing-partition-for-run-step                 | Dieser Fehler zeigt an, dass die im Ausführungsprofil angegebene Partition nicht gefunden werden kann. Stellen Sie sicher, dass die Partition nicht gelöscht oder umbenannt wurde.  |

## <a name="export-errors"></a>Exportfehler

| Fehler                    |Beschreibung     |
|--------------------------|------------|
